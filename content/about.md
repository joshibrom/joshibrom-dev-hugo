I'm a Web Applications Developer III at Texas A&M University-San Antonio, where
I assist in leading web development for ITS Web Services, recently
re-engineering a core data pipeline for one of our application that dropped
processing times from several minutes to seconds.  

My interests sit at the intersection of web infrastructure and systems
programming. I'm drawn to problems where performance actually matters:
concurrent I/O, memory-mapped data, and zero-allocation parsing. Zig and Rust
are where I spend most of my personal time right now.  

I also think carefully about the tools developers use--static site generators,
build systems, deployment workflows--and run my own self-hosted infrastructure.
